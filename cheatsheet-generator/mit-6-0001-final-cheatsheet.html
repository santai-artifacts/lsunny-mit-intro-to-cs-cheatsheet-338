Testing & Debugging

Test Design

  • Normal cases: Expected inputs
  • Boundary: Empty, single, max values
  • Error cases: Invalid inputs

Debugging

  • Print statements → trace execution
  • Binary search → narrow down bug location
  • Read code carefully → logic errors

Aliasing & Mutability

list1 = [1, 2, 3]
list2 = list1 # SAME OBJECT!
list1.append(4)
print(list2) # [1,2,3,4]!

Avoiding Aliasing

list3 = list1.copy() # or list1[:]
list3 = list(list1) # new list
⚠️ Side Effects make code unpredictable. Return values instead!

Problem-Solving Recipe

  1. Understand the problem
    • Inputs? Outputs? Constraints?
  2. Design solution
    • Decompose into parts
  3. Implement cleanly
    • Meaningful names, docstrings
  4. Test thoroughly
    • Normal, boundary, error cases
  5. Optimize if needed
    • Measure, find bottleneck, improve

🚨 Common Mistakes

  • Assigning mutable defaults in functions
  • Aliasing when you meant copy
  • Off-by-one errors in loops
  • Forgetting base case in recursion
  • Side effects instead of returns
  • Modifying global without declaring
  • Not testing edge cases (empty, single item)
